FPGA-Clock Network

发表于 2020-04-10 更新于 2020-08-14 分类于 可编程逻辑系统设计与 FPGA 技术 阅读次数： Valine：
本文字数： 4.3k 阅读时长 ≈ 4 分钟

介绍 FPGA 里时钟信号的产生与传输；

分布式时钟网络

时钟分布式网络构成

时钟分配网络由金属走线和缓冲器组成，将时钟信号传播到所有时钟元件：

  • 使用中继器缓冲时钟信号并均衡延迟；

PLL（Phase Locked Loop，锁相环）

PLL（锁相环）作为时钟发生器，是所有时钟信号的来源；

两种时钟生成方式

基于 PLL 的时钟生成

  • 宽频率合成范围
  • 精细分辨率的相位选择
  • 有效降低抖动

基于 DLL 的时钟生成


DCM (Digital Clock Manager)，即：数字时钟管理器，提供：

  • DLL（Delay Lock Loop）：延迟锁定循环
  • DFS（Digital Frequency Synthesis）：数字频率合成
  • DPS（Digital Phase Shifter）：数字移相器

Clock uncertainty

Clock uncertainty：$\Delta = skew \pm jitter$

Clock skew

定义：每个触发器的时钟到达时间的确定性差异

产生原因：主要是由于时钟树/网格的不平衡引起的；（Buffering 时钟信号可以减少但不能消除 skew）；

Clock jitter

定义：每个触发器的时钟到达时间的确定性或随机性差异；

产生原因：由管芯上的工艺，电源噪声，温度变化，PLL 抖动，串扰等引起；

时钟分布拓扑结构

主要有两种结构：tree-based structure、grid/mesh-based structure

对于同步操作，时钟到达时钟控制的元件，必须满足如下要求：

  • 极低的时钟 skew（偏斜）：
    • 确保最小的内部 hold-time 问题
  • 低时钟 jitter（抖动）：
    • 允许最高性能
  • 占空比保持：
    • 对双倍数据速率（DDR）应用非常重要
  • 低插入延迟：
    • 对同步输入和输出接口很重要

基于树的结构

  • 广泛应用于 ASICs（专用集成电路，Application Specific Integrated Circuits）；
  • 要保证时钟信号同时送往每个末梢；
  • 优点：
    • 在接线、电容、功耗上，低开销；
    • 易于进行时钟门控（不用时，可以随时关闭）；
  • 缺点：
    • FF 分布不均等导致难以平衡路径延迟（难以保证时钟信号同时到达末梢）；
    • 对 PVT（过程，电压，温度）变化敏感；

基于网格/网格的结构

  • 结构特点：
    • $n \times n$ 的均匀网格；
    • 由 $k \times k$ 个 buffers 组成的分布式数组驱动网格;
    • Buffers 由全局 H 树驱动;
    • Flip-flops 触发器直接连接到最近的网格段;
  • 用于现代微处理器；
  • 优点：
    • 非常适合低 skew（偏斜：时钟到达不同位置的时差）；
    • 强大的变化能力；
  • 缺点：
    • 更大的布线面积，大的负载电容，更多的功耗；
    • 难以分析：循环和冗余

对比总结

Topologies 应用 特点 Advantage Disadvantage
H-tree Altera, Stratix 树状的高度对称的架构 可提供出色的时钟偏斜控制 需要更多的路由资源来实现所需的对称性；
更大的插入延迟；
Rip-spine Xilinx, Virtex 4~7 包含类似于鱼骨头的主网络和子网络 易于实现 子分支中存在较大的时钟偏斜
Hybrid Altera, Stratix V H-树和 Rip-spine 结构的结合；
将 H-tree 作为主网络，将 Rip-spine 作为子网络；		 综合 综合

最坏情况下的偏斜约为数百皮秒；

控制范围 特点 优点
Global clock network 时钟驱动整个芯片
Regional clock network 时钟仅适用于他们进入的特定区域 较低的时钟插入延迟和偏斜

三个网络阶段

第一阶段


时钟网络的第一级以可编程方式将芯片外围的时钟源连接到每个区域的中心:

  • 第一阶段包括两个并行网络：一个用于全局时钟，一个用于本地时钟
  • 将全局时钟信号总数表示为：$W_{global}$
  • 全局时钟网络从每一侧的每个 $\frac{n_{source}}{4}$ 个潜在时钟源中选择 $\frac{W_{global}}{4}$ 个信号
  • 所有 $W_{global}$ 信号均通过专用线路路由到芯片中心

优势：由于信号源均来自芯片外围，因此信号源与芯片中心之间的连接几乎不会或根本不会产生偏斜;

  • 在某些来源来自芯片内部的架构中，可以调整驱动器的大小以使歪斜最小化

通过带有 ny_region 肋的脊柱和肋骨分配网络，从芯片的中心开始将所有 $W_{global}$ 时钟分配到所有区域的中心；

尽管 H 型树拓扑具有较低的偏斜度，但将这种拓扑网格划分到具有任意行数和列数的平铺 FPGA 上更加困难；

第二阶段


时钟网络第二级以可编程方式将时钟信号从每个区域的中心连接到该区域内的逻辑块：

  • 每个第二级网络的输入均由第一级的 $W_{global}$ 全局时钟和 $W_{local}$ 本地时钟组成：
    • 这些时钟使用脊柱和肋骨拓扑分布
    • 脊柱包含 $W_{global}+W_{local}$ 导线
  • 在每一行中，都使用 concentrator（集中器）网络从脊柱中选择任意一组 $W_{rib}$ clocks：
    • 这些时钟通过局部肋骨分配到该行中的逻辑块
  • 行中的每个逻辑块都通过另一个集中器网络连接到肋，集中器再从肋骨中选择任意一套 $W_{lb}$ 时钟

第三阶段


最后，第三级网络将 $W_{lb}$ 逻辑块时钟可编程地连接到逻辑块内的 N 个 logic elements:

  • 为了给群集工具提供灵活性，我们假设逻辑块的时钟引脚使用完整的交叉网络连接到逻辑元件（在该逻辑块内）的时钟引脚；
    • 因此，N 个逻辑元素中的任意一个都可以由任意 $W_{lb}$ 个逻辑块时钟提供时钟
  • 注意：这只是时钟网络中使用完整 crossbars（交叉开关）的阶段，其他阶段使用集中器交叉开关来减少开关数量；

时钟网络中的功耗降低

时钟网络由于其高扇出和切换频率而在FPGA中占很大一部分功耗。

  • 约占 FPGA 总功率的 20-39％：

主要是两种技术：by spine reduction、by clock gating

逻辑块的功耗感知布局

“功耗感知布局”：一种以功耗为导向的布局方法，来减少 spines 的数目；

  • 减少时钟网络上的互连资源使用
  • 考虑优化时钟功耗的布局

“脊柱减少”：

  • 将相同时钟的 CLB 放在一起；
  • 实际例子：

将时钟功耗添加到成本函数中：

  • $Cost = \alpha \cdot WireLength + \beta \cdot ClockPower$
  • WireLength：两个 CLB 之间互连线的长度；
  • 通常交换 CLB 时，spines 减少了，但是 WireLength 长度也会随之增加（时钟功率的降低是以增加电线段数为代价的）；
  • 通过 Trade off 得到 minimum cost：spine reduction 60%，WireLength increase 4%

Clock gating

主要思想：在设计中为那些空闲区域禁用时钟，以最大程度地减少信号转换，从而减少动态功耗；

已广泛用于 VLSI（超大规模集成）的电源优化；

门控时钟主要用在 ASIC（专用集成电路），为什么在FPGA设计中不建议使用时钟门控？

  • 因为时钟门控将逻辑放在 FPGA 内部的时钟网络上；
  • 时钟具有自己优化的低偏斜总线，在时钟源和时钟线之间引入逻辑将导致欺骗斜总线无法轻松使用；

例子：Virtex V

  • 全局时钟缓冲器（BUFGs）从外部或内部源接收时钟信号，并将这些信号馈入专用的全局时钟互连网络：
    • 全局时钟互连网络以低偏斜，低抖动和低功耗在整个 FPGA 中分配时钟信号；
    • 每个芯片包含32个全局时钟缓冲器，因此可以支持设计中存在 32 个全局时钟信号；
    • 架构上的限制限制了设备上每个触发器都具有 32 个单独的全局时钟信号的可能性；
    • 时钟区域高 20 个CLB，水平跨过 Virtex-5 芯片的一半；
    • 最多 10 个全局时钟信号可以馈入任何给定区域；
    • 一个区域内的 10 个全局时钟可以从设计中存在的（最多）32 个全局时钟信号中的任何一个中选择；
  • Virtex-5 内置的两种时钟使能机制可用于实现时钟门控：
    • SLICE 内的寄存器具有时钟使能引脚，允许以精细的粒度实现时钟使能；
      • SLICE 中的所有寄存器必须接收相同的时钟使能信号；
    • 将时钟信号驱动到专用时钟互连网络中的全局时钟缓冲器（BUFG）具有使能引脚
      • 当 BUFG 与启用一起使用时，称为 BUFGCE；
      • 使能引脚可以由来自 I/O 的信号或内部信号驱动；
      • 取消使能时，BUFG 驱动的时钟网络将变为静音，并且不会消耗任何动态功率；
    • 一个主要的观察结果是：
      • 使用 SLICE 上的时钟使能引脚并不能消除全局时钟网络上的切换，因此不能节省全局时钟网络上的动态功耗；
      • 但是，使用 BUFG 上的时钟使能引脚将停止在 BUFG 驱动的全局时钟网络上的所有切换；

时钟网络的设计

拓扑建模

FPGA 本身几何分布的参数：

  • $n_x$：BLK 列的数目
  • $n_y$：BLK 行的数目
  • $N$：每个逻辑块的逻辑元素（LE）数

FPGA 时钟网络的参数：

  • $n_{x_region}$：X 维时钟区域数目
  • $n_{y_region}$：Y 维时钟区域数目
  • $n_{source}$：潜在时钟源的数量
  • $W_{glb}$：一次可传送到 fabric 的 global 时钟数目
  • $W_{loc}$：一次可传送到 fabric 的 local 时钟数目
  • $W_{rib}$：一次可以路由到一个区域的一行中的所有逻辑块的时钟数目
  • $W_{lb}$：可以连接到每个逻辑块的时钟数目
  • $W_{le}$：可以连接到 LE 的时钟数目

时钟网络组件

  • Spine-Rib switch
  • Rib-LB switch
  • LB-LE switch
  • clock buffers

正是因为这些 switchs，实现了时钟网络的可编程性；
